Yea-Ree Chang

Website: University of Michigan Notable Alumni

Role: Quality Assurance

Evaluation Checkpoint Checked Notes
Run through each page with the WAVE Chrome Extension. This page received 11 errors from WAVE. The errors consisted of five headings with no content and six links with no text to describe their purpose.
Users should be able to navigate through content using their keyboard. The tabbing is promising; it allows the user to skip to the content or the navigation at the beginning and it tabs in a rational order. However, although the user is able to tab through all other links and buttons without getting trapped in any element, she does not actually have the access to the real content of this page â€“ each category of the alumni.
Users should be able to navigate content using a screen reader. Everything is readable by a screen reader except for the social media icons that resulted in the six WAVE errors.
The general architecture and hierarchy of the content should make sense.

The general architecture and hierarchy of the content does make sense. There is clear typographic hierarchy with clear headers and there are interactive elements that also help like hovering effects over clickable elements.

The only issue that I ran into while navigating through this website is that the expandable category boxes are a bit tricky to use. There is a distinguished expand button far away from the text and in a color that does not contrast very well with the background. If this button is not used, the user has to click a couple of times on a particular area of the box itself (not its entirety) for it to open
Charts and images should all have alt-text so that users with screen readers or users on a slow connection will still be able to understand the images. The one image on this website has alt-text.
Decorative images should not be visible to screen readers. [Decorative images, as in they do not add additional information.] This is not applicable since there are no decorative images present.

Suggestions

  1. My first recommendation is to add ARIA labels to the social media links. This would not only resolve the WAVE errors, but this would also make this website more accessible to users, especially those who have a hard time seeing icons and need to use a screen reader.
  2. The HTML of this website also is not properly written. The web designer should pay more attention to this, instead of writing empty tags that are meant to have text in it. The designer should also run the website through the validator because there do exist errors which show that the HTML was not written in the most efficient fashion.
  3. The boxes for each category of alumni have some room for improvement. The clickable area does not expand throughout the entirety of each box and the button explicitly designed for this purpose is far away and not very noticeable with its small size and low-constrast color. To resolve this, I would either make the button very obvious and place it next to the text or just expand the clickable area. The designer would also need to make it so that it is navigable by keyboard.
  4. I would imagine that using the search button might be a bit tricky for some users. Not only is it small, but the cursor does not change to reflect that it is a clickable element that does have an action attached to it. The search bar is also very small and might be hard to read for some.
  5. There also does not exist a nav tag for the navigation links in the header. The designer should consider including this as an important HTML landmark that helps users with assistive devices to further define specific areas of the website.
